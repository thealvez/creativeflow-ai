## 🛠️ Technical Details

### **Terraform Structure**
```
terraform/
├── main.tf          # Vercel provider + resources
├── variables.tf     # Input variable definitions
├── outputs.tf       # Output values
├── terraform.tfvars # Development values
├── .gitignore       # Exclude state files
└── README.md        # Documentation
```

### **Key Resources**
```hcl
resource "vercel_project" "frontend" {
  name             = "creativeflow-frontend"
  framework        = "nextjs"
  root_directory   = "packages/frontend"
  node_version     = "20"
  git_repository = {
    type = "github"
    repo = "thealvez/creativeflow-ai"
  }
}

resource "vercel_deployment" "frontend" {
  project_id = vercel_project.frontend.id
  reference  = "master"
  production = true
}
```

### **Environment Variables in Terraform**
```hcl
variable "vercel_token" {
  description = "Vercel API token"
  sensitive   = true
  # Set via: export TF_VAR_vercel_token="..."
}

resource "vercel_project_environment_variable" "api_url" {
  project_id = vercel_project.frontend.id
  key        = "NEXT_PUBLIC_API_URL"
  value      = var.api_url  # Production URL
  environments = ["production"]
}
```

## 📌 Notes

- Terraform Cloud can auto-approve and auto-deploy
- Rollback via `terraform destroy` then `terraform apply`

## 👨‍💻 Developer Handoff

**Owner:** @devops (Gage)
**When ready:** Execute terraform apply
**Prerequisites:**
- Vercel account created
- GitHub repository exists
- Vercel API token generated

**How to start:**
1. Create terraform/ directory
2. Write main.tf, variables.tf, outputs.tf
3. Set TF_VAR_vercel_token environment variable
4. Run `terraform init`
5. Run `terraform plan` (review)
6. Run `terraform apply` (deploy)
7. Commit terraform/ directory
8. Test: Push to GitHub master → Verify deployment

**Estimated Effort:** 2-3 hours
