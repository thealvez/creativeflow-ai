# Story 1.1.2: Monorepo Structure & Package Setup

**Epic:** EPIC 1 — Foundation
**Status:** 📋 Draft
**Story ID:** 1.1.2
**Points:** 5
**Complexity:** Medium
**Sprint:** Sprint 1

## 🎯 Story Summary

Setup complete monorepo structure with Node.js workspaces, configure package.json files for frontend (Next.js), backend (Fastify), and agents (LangChain), and establish shared configuration files (TypeScript, ESLint, Prettier).

---

## 👤 User Story

**As a** developer joining the CreativeFlow AI project
**I want** a properly structured monorepo with clear package separation
**So that** I can navigate the codebase easily and understand the architecture from directory structure.

---

## ✅ Acceptance Criteria

### AC-1: Monorepo Structure Created
```
✓ packages/frontend/   — Next.js 15 frontend application
✓ packages/backend/    — Fastify backend API
✓ packages/agents/     — LangChain agent orchestrator
✓ Each package has own package.json with independent scripts
✓ Shared files in root (tsconfig.json, eslintrc, etc)
```

### AC-2: Package.json Workspace Configuration
```
✓ Root package.json defines npm workspaces:
  "workspaces": ["packages/frontend", "packages/backend", "packages/agents"]

✓ npm ci installs dependencies for all packages
✓ npm run dev runs all dev servers
✓ npm run lint/test/build works across all packages
✓ No circular dependencies between packages
```

### AC-3: Frontend Package Configured
```
✓ @creativeflow/frontend package created
✓ Dependencies: next, react, react-dom
✓ Dev dependencies: typescript, eslint, tailwindcss
✓ Scripts: dev, build, start, lint, typecheck
✓ Can run: npm run dev (from /packages/frontend)
```

### AC-4: Backend Package Configured
```
✓ @creativeflow/backend package created
✓ Dependencies: fastify, pg, redis
✓ Dev dependencies: typescript, eslint, tsx
✓ Scripts: dev, build, start, lint, typecheck
✓ Can run: npm run dev (from /packages/backend)
```

### AC-5: Agents Package Configured
```
✓ @creativeflow/agents package created
✓ Dependencies: @anthropic-ai/sdk, langchain
✓ Dev dependencies: typescript, eslint, tsx
✓ Scripts: dev, build, lint, typecheck
✓ Can run: npm run dev (from /packages/agents)
```

### AC-6: Shared Configuration Files
```
✓ tsconfig.json (root) — TypeScript configuration for all packages
✓ .eslintrc.json — Linting rules (root)
✓ .prettierrc.json — Code formatting (root)
✓ .gitignore — Node modules, build outputs, env files
✓ All packages inherit root configuration
```

### AC-7: Scripts Working End-to-End
```
✓ npm ci → Installs all workspace dependencies
✓ npm run lint → Lints all packages
✓ npm run typecheck → Type-checks all packages
✓ npm run build → Builds all packages sequentially
✓ npm run dev → Starts all dev servers
✓ No errors, warnings only for deprecations
```

### AC-8: Documentation Complete
```
✓ README.md updated with monorepo structure explanation
✓ DEVELOPMENT.md — Local setup instructions
✓ Package README files explaining each module's purpose
✓ TypeScript paths documented (if using path aliases)
```

- Real-time WebSocket client

### **Backend Package Purpose**

- OAuth token management

### **Agents Package Purpose**

- Claude API integration

- ESLint code quality rules
- Prettier formatting rules
- Shared type definitions (if needed)

## 📌 Notes

- All packages use ES modules (type: "module" in package.json)
- No monorepo tool needed (native npm workspaces)
- Shared scripts in root can delegate to individual packages
- Each package remains independently publishable (future)
- CI/CD will run from root (single build pipeline)
