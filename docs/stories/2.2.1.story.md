# Story 2.2.1: Create Creative Form — UI/UX Design (Figma)

**Epic:** EPIC 2.2 — Create Creative Form
**Status:** 📋 Draft
**Story ID:** 2.2.1
**Points:** 3
**Complexity:** Low
**Sprint:** Sprint 1

---

## 🎯 Story Summary

Design the "Create New Creative" form UI in Figma with all input fields, validation states, error messages, and success confirmation. Create responsive layout for desktop, tablet, and mobile.

---

## 👤 User Story

**As a** creator
**I want** a simple, intuitive form to describe my video idea
**So that** I can quickly submit it for automated generation.

---

## ✅ Acceptance Criteria

### AC-1: Form Layout Design (Figma)
```
✓ Figma file created: "CreativeFlow - Create Form"
✓ Desktop layout (1440px wide):
  - Header: "Create New Creative"
  - Instructions text
  - Form section with 4 input fields
  - Submit button (primary)
  - Cancel button (secondary)

✓ Tablet layout (768px wide):
  - Responsive columns
  - Touch-friendly buttons (min 44px height)

✓ Mobile layout (375px wide):
  - Single column
  - Full-width buttons
  - Readable text (min 16px)
```

### AC-2: Form Fields Designed
```
✓ Field 1: "Describe Your Idea"
  - Type: Textarea (multi-line)
  - Placeholder: "E.g., 'Productivity tips with pomodoro technique'"
  - Min 10 chars, max 500 chars
  - Character count indicator
  - Empty state indicator

✓ Field 2: "Select Platforms"
  - Type: Checkboxes
  - Options: TikTok, Instagram Reels, YouTube Shorts, Facebook, Twitter
  - Min 1 selection required
  - Icons for each platform
  - Error state if none selected

✓ Field 3: "Language" (Optional)
  - Type: Dropdown
  - Default: "English"
  - Options: English, Portuguese, Spanish, French, etc.

✓ Field 4: "Tone" (Optional)
  - Type: Radio buttons
  - Options: Educational, Entertaining, Inspirational, Funny
  - Default: Educational
```

### AC-3: Validation States Designed
```
✓ Empty state (initial):
  - All fields visible
  - Placeholders shown
  - Submit button disabled (gray)
  - No error messages

✓ Valid state:
  - All required fields filled
  - Submit button enabled (blue, hover effect)
  - Character count shows remaining

✓ Invalid state:
  - Missing required field highlighted in red
  - Error message below field: "Required"
  - Submit button disabled

✓ Loading state:
  - Submit button shows spinner
  - Button text: "Creating..."
  - Form fields disabled (read-only)
```

### AC-4: Success Confirmation Designed
```
✓ Success screen after submit:
  - Checkmark icon (animated)
  - Message: "Creative submitted! Processing video..."
  - Job ID displayed (for reference)
  - "View Progress" button → Job status page
  - "Create Another" button → Reset form
```

### AC-5: Error Handling States
```
✓ API error screen:
  - Error icon
  - Message: "Something went wrong"
  - Retry button
  - Support link

✓ Validation errors:
  - Red border on invalid field
  - Red text error message
  - Character count in red if exceeded
```

### AC-6: Accessibility Design
```
✓ Labels associated with inputs (for attribute)
✓ Proper ARIA roles (aria-label, aria-describedby)
✓ Keyboard navigation order logical
✓ Focus indicators visible (min 2px border)
✓ Color contrast meets WCAG AA (4.5:1 for text)
✓ Form announces validation errors to screen readers
```

### AC-7: Design System Consistency
```
✓ Colors:
  - Primary: #3B82F6 (Blue)
  - Success: #10B981 (Green)
  - Error: #EF4444 (Red)
  - Background: #FFFFFF (White)
  - Text: #1F2937 (Dark Gray)
  - Disabled: #D1D5DB (Light Gray)

✓ Typography:
  - Heading: 28px Bold (Desktop), 24px Bold (Mobile)
  - Label: 14px Bold
  - Body: 14px Regular
  - Helper: 12px Regular

✓ Spacing:
  - Input padding: 12px 16px
  - Field spacing: 24px
  - Form padding: 32px (Desktop), 20px (Mobile)

✓ Components:
  - Button: 44px height, 16px padding horizontal
  - Input: 40px height, rounded corners (4px)
  - Checkbox: 20px × 20px
```

### AC-8: Responsive Breakpoints
```
✓ Desktop: 1440px (default design)
✓ Tablet: 768px (optimized for iPad)
✓ Mobile: 375px (optimized for iPhone)
✓ All states tested at each breakpoint
```

## 🛠️ Technical Details

### **Form Submission Flow**
```
User fills form
    ↓
Click "Create"
    ↓
Frontend validation (JavaScript)
    ↓
API call: POST /api/creatives
    {
      "idea": "...",
      "platforms": ["tiktok", "instagram"],
      "language": "pt-BR",
      "tone": "educational"
    }
    ↓
Backend creates job
    ↓
Returns: { id, status, job_id }
    ↓
Frontend shows success → Redirect to status page
```

### **Character Counter Logic**
```
- Max: 500 characters
- Min: 10 characters
- Display: "Characters: 45/500"
- Color: Green if < 450, Yellow if 450-490, Red if > 490
```

### **Platform Selection**
```
- TikTok (priority)
- Instagram Reels
- YouTube Shorts
- Facebook Video
- Twitter Video
- Min 1 selection required
- Show warning if none selected
```
